# AGENTS.md

## 프로젝트 개요
- 프로젝트명: `AI assistant for Pack`
- 목적: 팩 건강보험 상품상세페이지를 본 고객에게 입력값을 받아 AI Agent가 3가지 추천 설계안을 제안하는 HTML 프로토타입을 단계적으로 구현한다.
- 현재 범위:
  - 1단계: 상품상세페이지 + 입력 CTA + 추천 바텀시트 + 로딩 + 추천 3안 + 상세 팝업
  - 2단계: 계산결과화면
  - 3단계: 청약시작 화면

## 응답 / 작업 원칙
- 기본 답변은 한국어로, 결론 먼저 짧게 말한다.
- 사용자가 빠른 수정을 원하면 설명보다 바로 구현한다.
- 기존 화면 스타일과 흐름을 최대한 유지하면서 최소 범위로 수정한다.
- 삼성생명 디지털보험 UX 문맥에서는 실제 전환, 이해도, 마찰 감소 관점이 우선이다.
- 사용자가 별도 요청하지 않으면 불필요한 리팩터링은 하지 않는다.

## 프로토타입 구조 규칙

### 1단계 `index.html`
- 상품상세페이지 상단 헤더는 고정형이다.
- 본문은 통이미지 기반 스크롤 구조다.
- 하단 CTA 영역은 실제 입력 가능한 컴포넌트다.
- 바텀시트 단계:
  1. 월 보험료 선택
  2. 주요 보장 선택
  3. 로딩 애니메이션
  4. 추천 설계안 3개
  5. 추천안 선택 후 상세 팝업

### 2단계 `result-1.html`
- 계산결과화면은 1단계에서 넘어온 보험료를 반영한다.
- 보험료/최대혜택가 표시는 오버레이가 아니라 실제 HTML 값으로 반영한다.
- 상단 헤더 `<` 동작은 진입 경로에 따라 다르게 복귀한다.

### 3단계 `apply-step-1.html`
- 청약시작 화면은 화면 크기에 맞춘 통이미지 기반 + 일부 실데이터 반영 구조다.
- 헤더 `<` 동작은 진입 경로에 따라 다르게 복귀한다.

## 입력값/상태 처리 규칙
- 생년월일 입력값으로 나이대와 표시 나이를 계산한다.
- 성별은 `남성`, `여성` 상태값을 사용한다.
- 추천 상세 팝업의 아래 문구는 입력한 생년월일/성별 기준으로 동적으로 반영한다.
  - 예: `보험기간 100세, 납입기간 20년, 남성/45세`
- 주요 보장 선택 상태값:
  - `암`
  - `뇌·심장 혈관질환`
  - `치매`
  - `균형`
- 추천 카드명 등 사용자 노출용 문구는 필요 시 상태값과 별도 표시용 라벨을 둔다.
  - 예: 추천카드에는 `뇌·심장 집중형`으로 표기

## 문구 / UI 카피 규칙
- 추천 팝업 타이틀은 현재 다음 패턴을 사용한다.
  - 추천 1: `강력한 {보장명}보장 선택!`
  - 추천 2: `균형잡힌 보장을 선택!`
  - 추천 3: `실속형 보장으로 선택!`
- 상세 팝업 주요 보장 리스트 위에는 `주요 보장 안내` 라벨형 구분선을 둔다.
- CTA, 바텀시트, 팝업 카피는 사용자가 수시로 미세 조정하므로 문자열 하드코딩 위치를 쉽게 찾을 수 있게 유지한다.

## 보장항목 규칙
- 보장항목은 `index.html` 내부 `coverageCatalog` 기준으로 관리한다.
- 현재 사용자 확정 문구를 우선 기준으로 사용한다.
- 보장금액 로직은 엑셀 `41개보장_설정값.xlsx` 의 `학습데이터` 시트 기준을 활용한다.
- 팝업의 보장항목 리스트는 추천 케이스별 연관성 높은 항목 중심으로 노출한다.
  - 암: 암 관련 보장 우선
  - 뇌심장: 뇌/심장 관련 보장 우선
  - 치매: 치매/간병 관련 보장 우선
  - 균형: 여러 카테고리 혼합

## 디자인 / 레이아웃 규칙
- 기본 폰트는 `Noto Sans KR`
- 전체 폰트는 과한 볼드 없이 비교적 정돈된 톤 유지
- 모바일 우선 화면
- iOS/안드로이드 브라우저 주소창 이슈 대응을 위해:
  - `viewport-fit=cover` 사용
  - `100svh` / `100dvh` 기반 높이 사용
  - 모바일에서 세로 중앙 정렬 대신 상단 기준 정렬
- 모바일 폭에서는 모서리 라운드/그림자를 제거하고 full-width로 보이게 한다.

## 애니메이션 규칙
- 바텀시트 전환은 좌우 슬라이딩
- 로딩 애니메이션은 로티 기반
- 로딩 문구는 순차 등장
- 팝업 하단 검정 말풍선은 약간 느리게 디졸브 등장
- 과한 모션보다 자연스럽고 안정적인 UX를 우선한다.

## 테스트 규칙
- 스모크 테스트 12개를 우선 기준으로 사용한다.

  - 추천안별 상세 팝업 타이틀/보장항목 연관성

  - `직접 설계할게요` → 2단계 이동
- 자동 테스트 스크립트:
  - `test\run-smoke-test.mjs`

## 수정 시 주의사항
- 사용자가 미세한 위치, 카피, 간격, 폰트 크기까지 자주 조정하므로 CSS 수치는 함부로 크게 바꾸지 않는다.
- 이미지 위 오버레이 방식은 모바일 실브라우저에서 깨질 수 있으므로 가능하면 실제 HTML 구조로 반영한다.
- `file://` 환경과 배포 URL 환경이 다를 수 있으므로, 모바일 브라우저 이슈는 배포본 기준으로도 고려한다.
- 이미 사용자가 승인한 흐름/문구를 다시 되돌리지 않는다.

## 추천 작업 방식
1. 먼저 현재 파일 상태를 읽는다.
2. 수정 범위를 가장 작은 단위로 정한다.
3. 필요한 경우 관련 화면(`index.html`, `result-1.html`, `apply-step-1.html`)만 함께 맞춘다.
4. 테스트 문서 또는 스모크 기준으로 영향 범위를 확인한다.
5. 배포 요청이 있으면 최신본으로 Vercel 재배포한다.
