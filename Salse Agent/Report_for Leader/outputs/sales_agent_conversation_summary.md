## 1. 최초 요청

사용자는 생명보험회사 디지털조직의 UX/AI 전문가 관점에서, 팩건강보험 가입자수 확대를 위한 3장짜리 보고용 페이지를 요청했다.

핵심 방향은 단순 AI 기술 도입이 아니라, 고객별 니즈에 맞게 탐색부터 가입 과정까지 경험을 개선하는 `Sales Agent` 제안이었다.

구성은 3가지 단계별 Agent로 정의됐다.

1. 맞춤설계 Agent
   - 고객의 나이, 성별, 지불 가능한 월보험료, 집중 보장 항목을 바탕으로 추천 설계안 3개 제안
   - 셀프계산 단계를 줄이는 장점

2. 설계이해 Agent
   - 어려운 보험용어와 보장항목 의미를 친근하게 설명
   - 고객이 설계를 직접 조절하고 최종 가입 판단을 하도록 도움

3. 청약완주 Agent
   - 청약 과정의 어려운 단계에서 도움말 제공
   - 이탈 시 임시저장, 이어서 청약 진행 방법 안내
   - 청약 완결까지 독려

## 2. 작업 지침 반영

프로젝트 지침에 따라 다음 기준을 적용했다.

- 기본 산출물은 한국어로 작성
- 단순 UI 제안보다 가입 전환을 막는 고객 마찰에 초점
- 여정 단계, 병목, 원인 가설, 검증 지표 순서로 구성
- 전환율뿐 아니라 고객 오해, 민원, 심의 리스크도 함께 고려
- 최종 확인용 이미지 산출물은 `output/images/`에 저장

## 3. PPT 3장 구성

최초 PPT는 다음 흐름으로 제작했다.

1. 전략 프레이밍
   - `Sales Agent`를 전체 서비스 타이틀로 설정
   - 고객이 보험을 포기하는 이유를 관심 부족보다 설계 탐색, 용어 이해, 청약 진행 마찰로 정의

2. Agent 서비스 구성
   - 맞춤설계 Agent, 설계이해 Agent, 청약완주 Agent를 고객 여정 순서로 배치
   - 각 Agent별 고객 입력, Agent 개입, 기대 효과를 정리

3. 실행과 측정
   - 내부 퍼널 데이터로 병목 확인
   - 맞춤설계 Agent MVP부터 시작
   - 주요 지표와 가드레일 지표 제시

반영 내용:

사용자가 작은 폰트가 잘 보이지 않는다고 하며 다음 수정을 요청했다.

- 최소 폰트 사이즈를 13pt 이상으로 조정
- 모든 폰트를 현재 사이즈보다 10% 확대
- 3페이지 단계박스 사이 화살표가 박스와 겹치지 않게 조정

반영 내용:

- 전체 폰트 크기를 10% 확대

- 3페이지 상단 단계박스 간격과 화살표 위치 재배치
- 확대 후 어색한 줄바꿈 일부 문구 보정
