너는 생명보험회사의 디지널조직에서 UX/AI 전문가(기획자)야 
3장짜리 보고용 페이지를 만들어야해 (피피티 또는 이미지)
아래 내용을 바탕으로, 각 페이지의 구성은 알아서 판단해줘

목적 : '팩건강보험'의 가입자수 확대
전략 : 단순히 AI기술을 도입하는것이 아니라, 고객별 니즈에 맞게 AI를 활용한 탐색부터 가입과정의 경험을 개선해주는 Agent 제안
방향 : 전체적인 서비스 타이틀은 Sales Agent 이며, 3가지 단계별 Agent(경험의 변화 유도서비스)로 구성 됨

1번 Agent (맞춤설계 Agent) : 고객의 나이, 성별, 지불가능한 월보험료, 집중적으로 보장받고 싶음 항목 중심으로 맞춤 설계안(추천3개) 제안, 셀프계산단계를 스킵할수 있는 장점
2번 Agent (설계이해 Agent) : 어려운 보험용어, 보장항목에 의미 등 고객이 직접 설계를 조절하고 싶어도, 이해하기 어려운 부분들을 친근하게 풀어서 설명해줌, 가입여부의 최종 판단에 긍정적 영향
3번 Agent (청약완주 Agent) : 청약과정의 어려운 단계에서 막히는 부분의 도움말을 제공하고, 이탈시 임시저장 또는 이후 이어서 청약 진행을 할수 있는 방법 등을 알려줌. 청약 완결까지 인내력있게 독려하고완수할 수 있도록 도와줌

아래의 방향(전략)을 간략히 표현하고, 각 단계별 Agent의 역할이 명료하게 전달되도록 표현해줘

우선, 팩건강보험을 가입하는 과정을 3개의 단계로 표현하고 (각 단계별로 화살표 처럼 흘러가는 느낌 표현)
각 단계와 단계사이에서 고객이 느끼는 허들을 간략히 함께 표현해줘

1단계. 상품상세화면 - 상품의 특징과 내용을 설명하는 화면 
2단계. 계산결과 화면 - 생년월일/성별 정보를 통해 보험료를 확인하는 화면
3단계. 청약시작하기 - 청약 프로세스 진행시작하는 화면
4단계, 청약완료 - 청약프로세스를 완결한 화면

1단계--> 2단계로 넘어갈때 허들 : 내가 원하는 목적/상황에 맞게 특장점을 바로 알기 어려워 보험료를 계산하고 싶은 생각이 크지 않음
2단계--> 3단계로 넘어갈때 허들 : 디폴트는 '인기' 기준으로 보험료 안내를 받지만, 와닿지 않음 왜 인기지?, 셀프로 변경한후 보장항목을 일일이 조절하려고 해도 어떤 보장이 어떤의미인지 용어도 알기 어렵고 일일이 보험료계산하는데도 오래걸림
3단계--> 4단계로 넘어갈때 허들 : 청약 단계가 길고, 인증이 번거로우며 중간에 병력사항등 구체적인 입력절차에서 부담을 느낌

[현재 퍼널률 데이터 참고]
1단계--> 2단계 : 계산률 12%
2단계--> 3단계 : 청약시작률 7%
3단계--> 4단계 : 청약완료률 9%
